## 功能特点

0. **下订单事务** — 下单流程使用事务保证一致性
1. **支付超时回滚** — 支付超时后回滚订单相关状态与资源
2. **全局异常处理器** — 统一异常处理与响应
3. **防重（幂等）** — 通过数据库唯一约束实现（前端按钮 loading 禁用）
4. **AOP 鉴权** — Token 鉴权与有效期校验
5. **商品分类缓存** — Spring Cache（也可用内存缓存）
6. **订单状态机** — 有限状态机管理订单状态流转（防止非法流转，并对「新加状态」开放）
7. **过滤策略模式** — 按不同策略过滤（菜系 / 风味 / 价格）
8. **超卖防护** — 防止库存超卖
9. **冗余存储（反范式）**
   - 违反 1NF：某字段存 JSON
   - 违反 2NF：订单明细表冗余存用户 ID
   - 违反 3NF：订单表冗余存储用户名称
10. **菜单生效日期与价格** — `valid_from_date` 存生效起始日及对应价格；同一菜品 3 个日期对应 3 条记录
11. **金额整数存储** — 金额字段按「分 × 100」存整数
12. **每日菜单预计算** — 每天零点定时预计算未来20天的次日菜单(支持预定未来20天的)，写入 `t_daily_menu`（当前所有有效菜品 ID）
13. **后门刷新接口** — 某天临时新增菜品时，调用该接口刷新当日可用菜单
14. **风味固定列** — `t_menu_item` / `t_user` 列不多，风味类型枚举为固定列，分别表示「用户需要什么风味」与「菜品提供什么风味」：
    - `flavor_spiciness` — 辣度(3档, 高90, 中50, 低10)
    - `flavor_taste` — 甜口 / 咸口(2类, sweet, salty)
    - `flavor_protein_level` — 高低蛋白(3档, 高90, 中50, 低10)
    - `flavor_fat_level` — 高低脂(3档, 高90, 中50, 低10)
15. **AI 问答推荐菜品**
16. **用户密码加密存储**
